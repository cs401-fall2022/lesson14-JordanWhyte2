TypeScript Tutorial for Javascipt Devlopers

General Typescript

Typescript is similar to Javascipt but adds another layer of a type system on top of it.

Typescript consistently checks that you assign language primitives like strings and numbers.


For example, if we initialize a string with Javascript:


let helloWorld = "Hello World";

Typescript will see the value is a string, and define its type as string.



Defining Types

Typesscipt can also create an object that has multiple types inside of it. These types can either be declared using an inferface or inferred using const:

const user = {
name: "Jordan",
id: 114172256,
};

interface User {
name: string;
id: number;
}

Using the interface is as simple as creating an object with the interface name as the type:

const user: User = {
name: "Jordan",
id: 114172256,
};

Both JavaScipt and TypeScript are object oriented programming, thus interface declaration can be used within classes



Creating types

TypeScript can be used to crate types by combining simple ones. This can be done with unions. A common use for this is to create a boolean type within your code.

type boolean = true | false;

Unions can also be used within functions to handle different types of parameters. The same function can be used to get the length of a string or an array. By using unions in parameters code can be simplified. Additionally functions can check the type of a parameter and produce different outputs depending.



A triple equals sign (===) can be used to check the type of an object:

if(typeof obj === "string"){
return obj; }

Generics can be used to provide variables to types. An array without a generic could contain anything, but a generic can describe what values are allowed to be stored within the array.

type StringArray = Array< string>;


Exceptions and Errors

Just like with other languages, TypeScript uses try, catch, and finally blocks. We can attempt to run a function and catch if that function throws an exception, then run the finally block every time.

try{
runTask();
}
catch(e)
{
console.log("Error",e);
}
finally{
console.log("Code complete");
}


Structural Type System

Typescript can check if objects are the same type by checking the shape of the values. Shape matching requires only a subset of an object's fields to be shape-matched.



Non-exception Failures

Javascipt can return undefined when referencing an object that does not exist. Typescript instead will throw an error about the variable not being defined. This allows for legitimate bug catching easier. Typescript is able to catch far more bugs than JavaScipt for things like typos.

const user= {
name: "Jordan",
age: 19,
};
console.log(user.location);

The code above would throw an error for the property location not existing inside of user.
